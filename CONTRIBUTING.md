# Contributing to Darshan Portfolio

Thank you for considering contributing to this project.

## 📌 Guidelines

- Keep the code clean and easy to understand
- Follow consistent naming conventions
- Maintain responsive design compatibility
- Test changes before submitting
- Avoid unnecessary dependencies and libraries
- Preserve the overall design consistency

---

## 🚀 Steps to Contribute

1. Fork the repository

2. Create a new branch

```bash
git checkout -b feature-name
```

3. Make your changes and commit them

```bash
git commit -m "Added new feature"
```

4. Push the branch

```bash
git push origin feature-name
```

5. Open a Pull Request

---

## 🛠️ Areas for Contribution

- UI/UX improvements
- Portfolio design enhancements
- Performance optimization
- Accessibility improvements
- Mobile responsiveness
- Code refactoring
- Bug fixes
- New portfolio features

---

## 📄 Code Style

- Use meaningful file and variable names
- Keep HTML semantic and well-structured
- Organize CSS into logical sections
- Write clean and modular JavaScript
- Avoid duplicate code
- Comment complex logic when necessary

---

## 🧪 Testing

Before submitting changes:

- Verify the website works correctly on desktop devices
- Check responsiveness on mobile screens
- Test all navigation links
- Verify GitHub, LinkedIn, and Resume links
- Ensure there are no console errors

---

## 💡 Suggestions

If you have ideas for improving the portfolio, feel free to:

- Open an issue
- Submit a pull request
- Share design or feature suggestions

---

Thank you for contributing and helping improve this portfolio project.
